# Project Overview
- Purpose: macOS-native sandbox app to manage multiple video-generation AI models (Sora/Veo/Kling, etc.), enabling prompt-based generation, gallery, and settings for keys and output folder.
- Current status: Phase 1 (Project Initialization); focus on setup, folder structure, basic UI; MVP is single-model flow (Kling) with prompt + aspect ratio + duration.
- Stack: macOS 15+, Swift 6, SwiftUI, SwiftData, URLSession async/await, Keychain Services; project is an Xcode project (OpenReelStudio.xcodeproj).
- Architecture: MVVM + Service Provider pattern; `VideoProviderProtocol` defines `startGeneration`/`checkStatus`; providers include `MockProvider` (actor, in-memory jobs) and `KlingProvider` (URLSession REST); `GenerationConfig` for aspect ratio/duration; `GenerationStatus` for processing/completed/failed.
- Data model: SwiftData `GenerationItem` (@Model) with id, prompt, modelId, status, timestamps, local/remote paths, metadata JSON, error message.
- Features implemented: SettingsView saves API keys to Keychain (`service`=com.vaddiction.OpenReelStudio; accounts `api_key_openai`, `api_key_kling_access`, `api_key_kling_secret`, `api_key_google`) and output folder bookmark via BookmarkStore; GeneratorView handles prompt/aspect ratio/duration, starts generation, polls provider, saves status/progress, and blocks generation until Kling credentials are set; Gallery shows GenerationItems grid with detail and delete; ContentView switches to KlingProvider when Access/Secret are present (JWT bearer token) and falls back to MockProvider otherwise.
